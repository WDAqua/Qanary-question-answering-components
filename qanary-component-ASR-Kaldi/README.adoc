= ASR Kaldi

toc::[]

== Intention

The component get the voice signal and transforms it to the textual representation which is saved in a standardized way to the Qanary triplestore.

== Behavior

As a Qanary component, it reads the question and the annotations produced by upstream components from the Qanary triplestore (the shared process memory), performs its task — transcribe a spoken question into its textual representation — and writes the result back as standard Qanary (Web Annotation / `qa:`) annotations. See <<Input specification>> / <<Output specification>> below for the exact annotation contract.

== Characteristics

* *Task type:* Automatic Speech Recognition (ASR).
* *Offline build:* builds and (where present) tests run offline.

== Implementation

* *Language:* Java (JDK 21).
* *Major frameworks:* Spring Boot 3.5 (`spring-boot-starter-parent`); the Qanary component framework `eu.wdaqua.qanary:qa.component` 4.0.0 (transitively Spring Boot Admin client + Apache Jena 5); Apache HttpClient 5.
* *Main component class:* `src/main/java/eu/wdaqua/qanary/component/kaldi/asr/SpeechRecognitionKaldi.java` (extends `eu.wdaqua.qanary.component.QanaryComponent`).
* *Tests:* 2 test class(es) under `src/test/java/` (`QanaryServiceControllerTest.java`, `SwaggerUiAvailabilityTest.java`).

== Build and run

Targets the current Qanary framework 4.0.0 (Spring Boot 3, Java 21). Install the
framework first (`mvn install` in the https://github.com/WDAqua/Qanary[Qanary]
repository), then build with JDK 21:

[source,bash]
----
JAVA_HOME=/path/to/jdk21 mvn clean package
docker build --build-arg JAR_FILE=target/qanary-component-asr-kaldi.jar -t qanary/qanary-component-asr-kaldi .
----

== Input specification

Voice signal

== Output specification

Textual representation of a question in a standardized way.
