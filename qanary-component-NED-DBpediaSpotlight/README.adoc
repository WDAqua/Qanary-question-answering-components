= NED DBpedia Spotlight

== Description

Receives a textual question, forwards it to DBpedia Spotlight API and writes result in JSON format.

== Input specification

Not applicable as the textual question is a default parameter

== Output specification

[source, ttl]
----
@prefix qa: <http://www.wdaqua.eu/qa#> .
@prefix oa: <http://www.w3.org/ns/openannotation/core/> .

<urn:qanary:output> a qa:AnnotationOfInstance .
<urn:qanary:output> oa:hasTarget [
    a   oa:SpecificResource;
        oa:hasSource    <urn:qanary:myQanaryQuestion> ;
        oa:hasSelector  [
            a oa:TextPositionSelector ;
            oa:start "0"^^xsd:nonNegativeInteger ;
            oa:end  "5"^^xsd:nonNegativeInteger
        ]
    ] .
<urn:qanary:output> oa:hasBody <urn:dbr:Resource> ;
    oa:annotatedBy <https://api.dbpedia-spotlight.org/en/annotate> ;
    oa:annotatedAt "2001-10-26T21:32:52"^^xsd:dateTime ;
    qa:score "0.5"^^xsd:decimal .
----

== Build and run

This component targets the current *Qanary framework 4.0.0* (Spring Boot 3,
Java 21). Install the framework first (`mvn install` in the
https://github.com/WDAqua/Qanary[Qanary] repository), then build clean:

[source,bash]
----
JAVA_HOME=/path/to/jdk21 mvn clean package
java -jar target/qanary-component-ned-dbpediaspotlight.jar
----

Docker:

[source,bash]
----
mvn clean package
docker build --build-arg JAR_FILE=target/qanary-component-ned-dbpediaspotlight.jar \
  -t qanary/qanary-component-ned-dbpediaspotlight .
----

Configuration is in `src/main/resources/config/application.properties` (the
DBpedia Spotlight endpoint, confidence, SSL handling, and the usual
`server.port` / `spring.application.name` / `spring.boot.admin.url`).

== Tests

`mvn test` runs offline unit tests (JSON-response parsing with a mocked service
fetcher, plus the framework cache test). The Spring Boot Admin client failing to
reach a non-running pipeline is logged but non-fatal.
