= Qanary component: QC-AnswerTypeClassifier

toc::[]

== Intention

Classify the question (e.g. its expected answer type).

== Behavior

As a Qanary component, it reads the question and the annotations produced by upstream components from the Qanary triplestore (the shared process memory), performs its task — classify the question (e.g. its expected answer type) — and writes the result back as standard Qanary (Web Annotation / `qa:`) annotations.

== Characteristics

* *Task type:* Question Classification (QC).
* *External service:* http://dbpedia.org/ontology/
* *Offline build:* builds and (where present) tests run offline.

== Implementation

* *Language:* Java (JDK 21).
* *Major frameworks:* Spring Boot 3.5 (`spring-boot-starter-parent`); the Qanary component framework `eu.wdaqua.qanary:qa.component` 4.0.0 (transitively Spring Boot Admin client + Apache Jena 5); Gson; Apache HttpClient 5.
* *Main component class:* `src/main/java/eu/wdaqua/qanary/component/answertypeclassifier/qc/AnswerTypeClassifier.java` (extends `eu.wdaqua.qanary.component.QanaryComponent`).
* *Tests:* 2 test class(es) under `src/test/java/` (`QanaryServiceControllerTest.java`, `SwaggerUiAvailabilityTest.java`).

== Build and run

Targets the current Qanary framework 4.0.0 (Spring Boot 3, Java 21). Install the
framework first (`mvn install` in the https://github.com/WDAqua/Qanary[Qanary]
repository), then build with JDK 21:

[source,bash]
----
JAVA_HOME=/path/to/jdk21 mvn clean package
docker build --build-arg JAR_FILE=target/qanary-component-qc-answertypeclassifier.jar -t qanary/qanary-component-qc-answertypeclassifier .
----
